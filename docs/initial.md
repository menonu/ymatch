I am starting a new project called ymatch. The purpose of this project is to provide users with a merchandise trading platform. Below are the use cases. Please create a detailed plan and an AGENTS.md file based on this information.

Use Cases:

Users can create arbitrary event groups.

Event groups are created with names such as "Yukari live 2025 winter," and merchandise information is linked to the event group.

Merchandise information contains data such as "Photo 01, Photo 02, ... Photo 30," and is associated with names and photos (optional).

Users can freely browse event groups and manage the inventory of merchandise they own based on that merchandise information.

Users can advertise trade matching requests based on merchandise they have in surplus and merchandise they are missing/want.

Based on this trade information, matching is performed to facilitate exchanges between users.

Since merchandise requires physical exchange, the platform will include simple messaging app features to allow sending/receiving messages and location information during the exchange.

Users access the platform via Android smartphones or iPhone apps.

A central server will be used as the backend. The server must be capable of running both locally (for testing) and in the cloud.
